## 🔴 미완료 / 다음 세션 필수 작업

### 1. 상품목록 하위카테고리 버튼 미표시 (최우선)

**증상:** `?cid=UUID`로 카테고리 진입 시 하위카테고리 버튼이 표시되지 않음  
**원인 분석:** `page.tsx`에서 `childCategories` 필터링이 실패 → 빈 배열 전달  
**의심 원인:** API 응답의 카테고리 필드 구조가 `CategoryInfo` 인터페이스와 불일치 가능

```
확인된 데이터:
- currentCategory: 정상 (식품/건강 id=b0d5bf61-af2b-4b45-9bf6-4e8ccaec94f6)
- childCategories: [] (빈 배열 — 과일/채소 등이 와야 함)
- products: [] (상품도 비어있음 — category_id 파라미터 전달 문제 가능)
```

**수정 포인트:**
- `ciseco-nextjs/src/app/[locale]/(shop)/(other-pages)/products/page.tsx`
- `fetchAllCategories()` API 응답에서 실제 필드명 확인 필요
- `c.parent_id === currentCategory.id` 비교 로직 검증
- `fetchProducts(cid)` 에서 category_id API 파라미터 올바르게 전달되는지 확인

**API 직접 확인 방법:**
```
GET https://kn541-production.up.railway.app/categories
→ items[].id 타입, items[].parent_id 타입 확인
GET https://kn541-production.up.railway.app/products?category_id={uuid}
→ 상품이 실제로 반환되는지 확인
```

## 핵심 규칙 (KN541 개발 규정)

- **URL 파라미터:** 카테고리는 반드시 `?cid=UUID` 사용 (`category_code`의 `#;` 특수문자 금지)
- **locale:** 모든 내부 링크 `/ko/` prefix 필수 (`localePrefix: 'always'`)
- **데이터:** 모든 데이터는 FastAPI(Railway) 경유, Supabase 직접 연결 금지
- **SSR:** `localStorage` 접근 시 반드시 `isMounted` 패턴 사용
- **API 응답:** `{status, data: {items, total}}` 표준
- **로고:** Supabase Storage `brands/white_logo.png` — 같은 이름으로 교체 업로드하면 자동 반영
