# 쇼핑몰 장바구니 담기 — 일반 쇼핑몰 표준 검증 구현

> 2026-04-21 | 기획 창 → Cursor(쇼핑몰) 전달용
> 레포: kn541/kn541-shop (ciseco-nextjs)

---

## 현재 상태

장바구니 기본 기능은 이미 구현 완료:
- `cart-context.tsx` — localStorage 기반, 동일 상품+옵션 수량 합산, 배송비 계산
- `ProductActions.tsx` — 장바구니/바로구매 버튼, NcInputNumber 수량, 옵션 UI

**문제: 검증이 전혀 없음** — 옵션 미선택, 품절 상품도 담기 가능

---

## 구현 요구사항

### 1. 옵션 필수 선택 검증

상품에 옵션(색상/사이즈)이 있으면 반드시 선택한 뒤에만 장바구니 담기 가능.

```
[검증 로직]
- options 데이터에 color 옵션이 존재하는데 colorSelected가 비어있으면 → 차단
- options 데이터에 size 옵션이 존재하는데 sizeSelected가 비어있으면 → 차단
- toast.error('옵션을 선택해주세요') 표시
- 해당 옵션 영역 강조 (빨간 테두리 또는 흔들림 애니메이션)
```

### 2. 품절/단종 상태 차단

상품 상태가 판매중(ON_SALE)이 아니면 구매 불가.

```
[Props 추가]
ProductActions에 productStatus?: string 추가

[검증 로직]
- productStatus가 'SOLD_OUT' → "품절된 상품입니다" 표시, 버튼 disabled
- productStatus가 'DISCONTINUED' → "단종된 상품입니다" 표시, 버튼 disabled  
- productStatus가 'WAITING' → "준비중인 상품입니다" 표시, 버튼 disabled
- 버튼 색상을 회색으로 변경, cursor-not-allowed
```

### 3. 재고 수량 체크

수량 선택이 재고 초과하면 차단.

```
[Props 추가]
ProductActions에 stock?: number 추가 (null이면 무제한)

[검증 로직]
- NcInputNumber의 max를 stock 값으로 설정 (stock이 있을 때)
- 장바구니 담기 시 이미 장바구니에 있는 동일 상품 수량 + 새 수량 > stock이면
  toast.error('재고가 부족합니다. (현재 재고: N개)')
- stock이 0이면 품절 처리
```

### 4. 장바구니 최대 상품 종류 제한

```
- 장바구니에 최대 50종까지 담기 가능
- 초과 시 toast.error('장바구니는 최대 50개 상품까지 담을 수 있습니다')
```

### 5. 바로구매 검증

바로구매도 동일한 검증 적용 (옵션 체크, 품절 체크, 재고 체크)

## Cursor 프롬프트

```
[KN541 쇼핑몰 — 장바구니 담기 표준 검증 구현]

ProductActions.tsx에 일반 쇼핑몰 수준의 검증 로직을 추가해줘.

현재 파일: ciseco-nextjs/src/app/[locale]/(shop)/(other-pages)/products/[handle]/ProductActions.tsx

1. Props 추가:
   - productStatus?: string  (ON_SALE/SOLD_OUT/DISCONTINUED/WAITING)
   - stock?: number          (재고 수량, null이면 무제한)
   - hasColorOption: boolean  (색상 옵션 존재 여부)
   - hasSizeOption: boolean   (사이즈 옵션 존재 여부)

2. 검증 로직 추가 (handleAddToCart, handleBuyNow 공통):
   a) 옵션 필수 체크:
      - hasColorOption && !colorSelected → toast.error('색상을 선택해주세요')
      - hasSizeOption && !sizeSelected → toast.error('사이즈를 선택해주세요')
      - return (담기 중단)
   
   b) 품절/상태 체크:
      - productStatus !== 'ON_SALE' && productStatus !== undefined
        → 버튼 disabled + 상태별 메시지
   
   c) 재고 체크:
      - stock이 존재하고 stock > 0일 때:
        - 이미 장바구니에 있는 동일 상품+옵션의 수량 확인
        - 기존수량 + 새수량 > stock이면 toast.error 
      - stock === 0 → 품절 처리
   
   d) 장바구니 50종 제한:
      - items.length >= 50이고 신규 상품이면 toast.error

3. UI 변경:
   - 품절/단종일 때 버튼 비활성화 (bg-gray-400, cursor-not-allowed)
   - 품절 시 "품절" 텍스트 오버레이
   - NcInputNumber의 max를 stock 값으로 설정

4. page.tsx 수정:
   - ProductActions에 새 Props 전달
   - product 데이터에서 productStatus, stock 추출하여 전달
   - options에서 hasColorOption, hasSizeOption 판단하여 전달

완료 후 장바구니 담기 테스트:
- 옵션 미선택 → 에러 토스트
- 품절 상품 → 버튼 비활성화
- 수량 초과 → 에러 토스트
- 정상 담기 → 성공 토스트
```

## 완료 기준

- [ ] 옵션 있는 상품 → 미선택 시 "옵션을 선택해주세요" 토스트 + 담기 차단
- [ ] 품절 상품 → 버튼 비활성화 + "품절된 상품입니다" 표시
- [ ] 단종 상품 → 버튼 비활성화 + "단종된 상품입니다" 표시
- [ ] 재고 초과 → "재고가 부족합니다" 토스트 + 담기 차단
- [ ] 정상 담기 → 장바구니 토스트 + 수량 합산 정상
- [ ] 바로구매도 동일 검증 적용
