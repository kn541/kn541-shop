## 1. 배송비 표시 버그 수정 — 상품 상세 페이지

### 이슈
- `page.tsx`에서 배송비(`shippingFee`, `returnFee`, `deliveryDays`)를 최상위에서 구조 분해
- `adapters.ts`는 배송비를 `delivery.shipping_fee` 중첩 객체로 저장
- 결과: 모든 배송비 필드가 `undefined` → 무조건 "무료배송" 표시

### 추가 이슈
- `sc_type` 로직 오류: `sc === 1 → '무료배송'`이었는데 KMC sc_type=1은 **유료 고정배송비**
- KN541 DB 기준: sc_type=1=무료, 2=조건부무료, 3=유료건당, 4=유료수량별
- sc_type=4(유료, 3,000원)인 상품도 "무료배송"으로 표시되는 버그

### 수정 (`ciseco-nextjs/src/app/[locale]/(shop)/(other-pages)/products/[handle]/page.tsx`)
```tsx
// 수정 전 (버그)
const { shippingFee, returnFee, deliveryDays } = p  // undefined!

// 수정 후
const deliveryInfo    = p.delivery || {}
const shippingFee     = Number(deliveryInfo.shipping_fee ?? 0)
const scType          = Number(deliveryInfo.sc_type ?? 1)
const returnFee       = Number(deliveryInfo.return_fee ?? 0)
const deliveryDays    = Number(deliveryInfo.delivery_days ?? 3)

// shippingText 로직 수정
if (scType === 1 || shippingFee === 0) return '무료배송'  // sc_type=1 명시
```

## 4. GET /kmc/products — calculated_supply_price 필드 추가

### 이슈
어드민에서 KMC 상품 목록 조회 시 우리 DB의 마진 적용 공급가(`products.supply_price`)가 없어
KMC 원본 공급가(`kmc_products.supply_price`)만 보임 → 마진 확인 불가

### 구현 (`kmc_option_sync.py`)
```python
def _attach_calculated_supply_prices(db, items):
    """
    product_id 기준으로 products.supply_price를 200건씩 배치 조회
    LEFT JOIN 대체: product_id 없으면 None
    """
```

### 수정 (`kmc_sync.py` — VS Code 직접 수정)
```python
# GET /kmc/products return 전
_attach_calculated_supply_prices(db, items)  # in-place 수정
return {"status": "success", "data": {"items": items, ...}}
```

### 주의사항 (수정 과정에서 발견한 버그)
- 최초 구현 시 `items = _attach_calculated_supply_prices(db, items)` 로 작성
- 함수가 `None` 반환 (in-place) → `items = None` 이 되는 버그
- `items =` 제거로 수정 완료
