## 버그 수정 상세

### 1. 상품목록 하위카테고리 버튼 미표시 (✅ 해결)

**원인:** `/categories` API가 **tree 구조**로 응답 (depth=2는 depth=1의 `children` 배열 안에 포함).
`page.tsx`의 `fetchAllCategories()`는 API 최상위 `items`만 받아 flat 배열로 쓰고 있었기 때문에
`parent_id === currentCategory.id` 비교 시 depth=2 항목이 배열에 없어 항상 `[]` 반환.

**수정:** `flattenCategories()` 재귀 함수로 tree → flat 변환. 모든 depth 항목을 단일 배열에 수집.
아울러 `id`, `parent_id` 모두 `String()` 통일로 타입 불일치 방지.

```
수정 전: allCategories = [depth1_A, depth1_B, ...]  ← depth2 없음
수정 후: allCategories = [depth1_A, depth2_A1, depth2_A2, depth1_B, ...]  ← 전체 flatten
```

### 2. 모바일 사이드바 구버전 URL (✅ 해결)

**원인:** `aside-sidebar-navigation.tsx`가 `getNavigation()` 정적 데이터(템플릿 원본 `/collections/all` URL)를 사용.
`CategoryNavClient.tsx`(데스크톱)는 이미 `/ko/products?cid=UUID`로 수정되어 있었으나 모바일 사이드바는 누락.

**수정:** `getCategories()` API 동적 로드로 교체. depth1/depth2 구조를 `TNavigationItem` 형식으로 변환.
URL: `/ko/products?cid={UUID}` 방식 통일.

## 핵심 규칙 (KN541 개발 규정)

- **URL 파라미터:** 카테고리는 반드시 `?cid=UUID` 사용 (`category_code`의 `#;` 특수문자 금지)
- **locale:** 모든 내부 링크 `/ko/` prefix 필수 (`localePrefix: 'always'`)
- **데이터:** 모든 데이터는 FastAPI(Railway) 경유, Supabase 직접 연결 금지
- **SSR:** `localStorage` 접근 시 반드시 `isMounted` 패턴 사용
- **API 응답:** `{status, data: {items, total}}` 표준
- **카테고리 API:** `/categories` 응답은 **tree 구조** (children 포함). flat 배열 필요 시 `flattenCategories()` 사용
- **로고:** Supabase Storage `brands/white_logo.png` — 같은 이름으로 교체 업로드하면 자동 반영
